Project: Contact+ Pro

Contact+ Pro is a desktop address book application designed for students of NUS. The user interacts with it using a CLI, and it has a GUI created with JavaFX. It is written in Java, and has about 6 kLoC. This address book help ease communications among students, be it with new classmates or long-time friends in NUS.

Adding a person: add

Adds a person to the address book
Format: add n/NAME p/PHONE_NUMBER e/EMAIL a/ADDRESS b/BIRTHDAY [m/MAJOR]? [fb/FACEBOOK]? [r/REMARK]? [t/TAG]…​

End of Extract

Justification

Adding the birthday feature helps to personalise the address book allowing students to get to know each other better.

Adding the shorthand alias helps new users to use CLI with ease. It also reduces the amount of effort spend keying in the information.

Implementing the white theme offers a cleaner option for users to use.

Birthday Detail

A birthday detail is implemented under the Person class as Birthday. Users will add the birthday in along with the add command. It is a compulsory attribute like the Name, Phone, Email, Address of the Person class. It’s implementation is similar as well.

Design Considerations

Aspect: Should this be an optional attribute instead? Alternative 1(current choice): It is easier to implement it as an compulsory field. Pros: Adding the birthday details makes the AddressBook more personal. Cons: Users have no choice but to key in the birthday detail. Alternative 2: Make the birthday attribute optional. Pros: Allows the user freedom to decide what they want in the addressbook. Cons: Slightly more tedious to implement.
